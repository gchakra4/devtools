üîÑ Daily Development Workflow

1Work on Development Branch

All day-to-day development happens on the dev branch:

# Switch to main project
cd "D:\New folder\tryfix - Copy"

# Make sure you're on dev branch
git checkout dev

# Check status
git status

# Make your changes in VS Code...
# (Edit components, add features, etc.)

# Add all changes
git add .

# Commit with descriptive message
git commit -m "Added new booking feature"

# Push to dev branch (deploys to dev.yogique.life)
git push origin dev
üí° Result: Changes automatically deploy to dev.yogique.life for testing

2Test on Development Site

Visit dev.yogique.life and thoroughly test your changes:

  • Test new features
  • Check for bugs
  • Verify responsive design
  • Test on different browsers

3Deploy to Production

When everything works perfectly, merge to production:

# Switch to production branch
git checkout main

# Merge dev branch into main
git merge dev

# Push to production (deploys to yogique.life)
git push origin main

# Switch back to dev for next feature
git checkout dev
‚úÖ Result: Changes are now live on yogique.life

üöÄ Quick Reference Commands

Check Which Repository You're In:

# Show current directory
pwd

# Show git repository URL
git remote -v

# Show current branch
git branch

Switch Between Projects:

# Work on main app
cd "D:\New folder\tryfix - Copy"

# Work on devtools
cd "D:\New folder\devtools"

Switch Between Branches (Main App Only):

# Switch to dev branch (for development)
git checkout dev

# Switch to main branch (for production)
git checkout main

# See all branches
git branch -a

Undo Changes (Before Commit):

# Discard changes to specific file
git checkout -- filename.js

# Discard all changes
git reset --hard

Pull Latest Changes:

# Update from GitHub
git pull origin dev      # For dev branch
git pull origin main     # For main branch

‚ö†Ô∏è Important Guidelines

‚ùå Never commit directly to main branch!

Always develop on dev branch first, test thoroughly, then merge to main.

‚úÖ Best Practices:
  • Always pull latest changes before starting work
  • Commit frequently with clear messages
  • Test on dev.yogique.life before merging to production
  • Use descriptive commit messages (e.g., "Fixed booking form validation")

üÜò Troubleshooting

‚ùì I forgot which branch I'm on

git branch
# * shows your current branch

‚ùì I made changes on the wrong branch

# Save your changes temporarily
git stash

# Switch to correct branch
git checkout dev

# Apply your changes
git stash pop

‚ùì I want to undo my last commit

# Undo last commit, keep changes
git reset --soft HEAD~1

# Undo last commit, discard changes
git reset --hard HEAD~1

‚ùì Push was rejected (conflicts)

# Pull latest changes first
git pull origin dev

# Resolve conflicts in VS Code
# Then commit and push again
git add .
git commit -m "Merged latest changes"
git push origin dev
